# OpenCloud Music AI Agent - PRD v1.0
## 产品需求文档 第一版 - 概念与愿景

**版本**: v1.0  
**日期**: 2026-03-17  
**状态**: 概念设计阶段  

---

## 1. 产品愿景

### 1.1 核心理念
打造一个基于 OpenClaw 生态的个人音乐 AI Agent，实现跨平台音乐聚合、智能推荐、AI 创作的全流程音乐体验。

### 1.2 目标用户
- 多平台音乐使用者（Spotify + 网易云 + QQ音乐）
- 音乐收藏爱好者
- 需要个性化音乐推荐的深度用户
- 对 AI 音乐创作感兴趣的创作者

---

## 2. 功能需求概览

### 2.1 核心功能模块

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| 音乐搜索 | 跨平台歌曲搜索、歌手搜索、专辑搜索 | P0 |
| 播放控制 | 播放/暂停、上一首/下一首、音量控制、进度跳转 | P0 |
| 歌单管理 | 创建歌单、添加歌曲、删除歌曲、智能歌单生成 | P0 |
| 收藏系统 | 喜欢/取消喜欢、收藏专辑、关注歌手 | P1 |
| 听歌统计 | 播放历史、听歌时长、偏好分析 | P1 |
| 每日推荐 | 基于听歌习惯的每日歌曲推荐 | P1 |
| 音乐上传 | 本地音乐上传到云端/音乐平台 | P2 |
| AI 创作 | 基于 AI 的音乐生成、歌词创作 | P2 |
| 跨平台同步 | 歌单、收藏、播放状态多平台同步 | P2 |

### 2.2 交互渠道

- 语音输入（语音助手集成）
- 飞书（消息 + 卡片）
- Telegram（Bot）
- Discord（Bot）
- Web 界面

## 3. 平台 API 调研初览

### 3.1 Spotify
- **API**: Spotify Web API
- **文档**: https://developer.spotify.com/documentation/web-api
- **功能**: 搜索、播放控制、歌单管理、用户数据
- **限制**: 需要 Premium 账户支持部分功能
- **认证**: OAuth 2.0

### 3.2 网易云音乐
- **API**: 非官方 API（NeteaseCloudMusicApi）
- **GitHub**: https://github.com/Binaryify/NeteaseCloudMusicApi
- **功能**: 搜索、播放、歌单、评论、每日推荐
- **限制**: 非官方，需自行部署

### 3.3 QQ音乐
- **API**: 官方 API（有限开放）
- **文档**: QQ音乐开放平台
- **功能**: 基础搜索、歌单
- **限制**: 部分功能需申请权限

### 3.4 Apple Music
- **API**: MusicKit
- **文档**: https://developer.apple.com/documentation/musickit
- **功能**: 播放、搜索、歌单
- **限制**: 需要 Apple Developer 账号

## 4. 技术架构初案

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    交互层 (Channels)                        │
├──────────┬──────────┬──────────┬──────────┬─────────────────┤
│  语音    │  飞书    │ Telegram │ Discord  │    Web UI       │
└──────────┴──────────┴──────────┴──────────┴─────────────────┘
                            │
┌───────────────────────────┼─────────────────────────────────┐
│                      OpenClaw Agent                         │
│  ┌─────────────────────────────────────────────────────┐  │
│  │                  Music Skill                        │  │
│  │  ┌──────────┬──────────┬──────────┬──────────────┐  │  │
│  │  │ 搜索    │ 播放    │ 歌单    │  推荐/AI    │  │  │
│  │  └──────────┴──────────┴──────────┴──────────────┘  │  │
│  └─────────────────────────────────────────────────────┘  │
└───────────────────────────┬─────────────────────────────────┘
                            │
┌───────────────────────────┼─────────────────────────────────┐
│                      服务层 (Services)                      │
│  ┌──────────┬──────────┬──────────┬──────────────────────┐  │
│  │Spotify   │网易云    │QQ音乐    │ Apple Music         │  │
│  │Service   │Service   │Service   │ Service             │  │
│  └──────────┴──────────┴──────────┴──────────────────────┘  │
│  ┌──────────┬──────────┬──────────┬──────────────────────┐  │
│  │ 记忆    │ 统计    │ 推荐    │  AI 创作            │  │
│  │Service   │Service   │Service   │ Service             │  │
│  └──────────┴──────────┴──────────┴──────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 数据流
1. 用户通过任一渠道输入指令
2. OpenClaw Agent 解析意图
3. Music Skill 调用相应服务
4. 服务层与音乐平台 API 交互
5. 结果通过原渠道返回给用户

## 5. 记忆系统设计

### 5.1 核心记忆类型
- **听歌历史**: 歌曲ID、播放时间、播放时长、播放来源
- **收藏数据**: 喜欢的歌曲、专辑、歌手
- **歌单数据**: 用户歌单、智能歌单、播放列表状态
- **偏好画像**: 流派偏好、歌手偏好、时段偏好
- **统计数据**: 总听歌时长、每日听歌量、连续打卡天数

### 5.2 记忆存储
- 格式: JSON / SQLite
- 位置: `~/.openclaw/music-memory/`
- 同步: 可选云同步（加密）
